# Scene Patterns

Pick one shot type per image. Each pairs a setting, a device, and a way the project content appears on screen.

## Exhibition shots

### Wall-mounted kiosk

For: a single-station interactive — a simulator or chooser.

How: a vertical/landscape touchscreen on a gallery wall (optionally a small webcam on top), one to three kids reaching to touch a control. Screen shows the experience title and main UI (e.g. a simulator with charts and a "Water Temperature" slider). A companion tablet mounted below can show a secondary view (a map, data). Dim blue ambient, screen-lit faces.

### Round / curved touch table

For: a shared, explore-together moment — a species or ecosystem browser.

How: a circular or oval table-screen seen from a slight angle, a family or small group gathered around, one finger touching an element (a fish, a node) that responds with a subtle highlight/AR glow. Reef/data graphics on the table surface. Warm faces, soft museum light.

### Large curved panoramic screen

For: a flagship data experience — maps, live readouts, deep-sea visuals.

How: a wide, slightly curved screen filling the frame with the project's data UI (e.g. an ocean heatmap + charts, a "DataXplorer"-style tool), a student reaching toward it while an OceanX facilitator and another student watch. Deep-sea ambiance, projected sea life (jellyfish) glowing at the edges. Cinematic, dark, screen-lit.

### Companion tablet (supporting)

For: pairing a big exhibit with a handheld view, or a guided handoff.

How: a person holding a tablet that mirrors/extends the main screen's content; can be a second subject in any exhibition shot. The tablet shows a focused slice of the project (controls, a map, a detail view).

## Home shots

### Home desktop

For: at-home web/game use, "play it from anywhere".

How: a person at a desk by a window, the experience open on a monitor or laptop — a browser running the web app, or a game window (e.g. a pixel-art ocean game with an ecosystem-health bar and a depth readout). Cozy room, daylight, mug/plant/books. Over-the-shoulder framing shows both the user and the screen. A second app window (browser, notes) is fine for realism.

### Home tablet / couch

For: casual, relaxed use.

How: a person on a couch or at a kitchen table holding a tablet running the experience, soft daylight, calm. Optionally a sibling/parent nearby.

## Composing each shot

1. Lock the project content first — what is on the screen, exactly? (title, main visual, controls)
2. Choose setting + shot type that make that content believable in use.
3. Choose the cast and the one core interaction (touching a slider, pointing at the map, watching a chart change).
4. Set the light per setting (dim blue exhibition / warm daylit home).
5. Frame to read both the person and the screen.

## Freshness rule

The `assets/examples/` photos calibrate realism, lighting, and framing — not exact people or compositions. Vary the cast, angle, device, and which part of the project is on screen. Don't reproduce a specific example shot; stage a new believable moment around the project's real content.
