# Storyboard Patterns

Pick one layout per image. Don't blend several into a cluttered frame.

## Layout types

### Numbered step strip (default)

For: a web experience or web-game flow with a clear sequence — choose → place → recognize → transform → interact.

How: 3–6 white panels left → right, each with a navy numbered circle + bold title, teal block arrows between them, a stick-figure actor doing each step. This is the look of the OceanX example storyboards. Use it unless another pattern fits better.

### Before / after shift

For: an environmental or ecological change, a "what happens when…" beat, success vs. failure of an intervention.

How: 2–3 panels showing initial state → shift → consequence. Use light-blue washes for the healthy state and scattered navy "glitch/disruption" pixels for the degraded state. The actor reacts (curiosity → concern).

### Single exhibit scene

For: one big interactive moment — a panoramic screen, an AR tank, a kiosk wall.

How: one wide panel, a curved/large light-blue screen or AR volume, the actor in front pointing or reaching, sparse line-art sea life on the surface. Minimal labels.

### Interaction loop

For: how a person and the system talk to each other — input, response, feedback, chat-with-AI.

How: actor + device + screen + a rounded speech/thought bubble (heart, node-graph, or waves). Teal arrows can curve to show the loop. Good as the final panel of a strip.

### Ecosystem / system map

For: relationships in a food web, a data pipeline behind the experience, how parts connect.

How: 3–5 line-art nodes (creatures, modules) connected by thin lines; the actor stands at one node doing the key action. Keep it sparse — not a dense architecture diagram.

### Game-flow storyboard

For: an educational web game — levels, screens, win/lose states, progression.

How: numbered panels showing each screen/level as a simple framed scene; teal arrows for progression; the actor (or a hand/cursor) interacting; a small score/heart/star motif for state. Don't draw a literal pixel-perfect UI.

## Designing each panel

1. Name the step in 1–3 words (becomes the navy title).
2. Decide the one core action the actor performs.
3. Pick the one key prop/screen/object — not three.
4. Add at most a couple of short labels; let the picture carry meaning.
5. Use light-blue wash only where there's a screen / water / AR / digital field.
6. Use the warm accent (a fish, a highlight) on at most 1–2 small elements.

## Freshness rule

Don't copy the example storyboards panel-for-panel. The coral-recognition and marine-timeline examples in `assets/examples/` exist to calibrate **style** — line weight, palette restraint, whitespace, how the actor participates — not to be reused.

Reinvent the beats for the experience in front of you:

- A different experience needs different steps, props, and actor actions.
- "Transformation" doesn't have to be an AR tank; it could be a kiosk, a map, a microscope view, a game screen.
- "Disruption" doesn't have to be navy pixels; it could be a draining tank, a fading reef, a tilting balance.

Same theme, new staging.
